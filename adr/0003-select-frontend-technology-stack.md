# ADR 0003: フロントエンド技術スタックの選定（Angular / TypeScript）

- **Status**: Accepted
- **Date**: 2026-05-03
- **Deciders**: System Architect

## 1. Context (背景)
SoE レイヤにおいては、ユーザー体験（UX）の向上と同時に、複数人での大規模開発に耐えうる「保守性」と「構造の強制力」が求められる。
フロントエンドは技術の移り変わりが激しいが、汎用的な基礎を学ぶためには、コンポーネント指向、DI（依存性の注入）、RxJS による非同期処理といった、高度なソフトウェア工学の概念を標準で内包しているフレームワークが必要である。

## 2. Decision (決定)
フロントエンドの主要スタックとして以下を採用する。

- **Framework**: **Angular**
  - 理由: 「フルスタック・フレームワーク」として、ルーティング、フォーム管理、HTTPクライアント等が標準化されており、開発者による構造のバラつきを最小限に抑えられる。また、Googleによる長期的なサポートが期待できる。
- **Language**: **TypeScript**
  - 理由: 静的型付けにより、バックエンド（Java）との型定義の共有や、大規模コードベースにおけるリファクタリングの安全性を確保する。
- **Library (Reactive)**: **RxJS**
  - 理由: イベントストリームを宣言的に扱うリアクティブプログラミングの習得。非同期処理のデファクトスタンダードなパターンを学ぶため。

## 3. Consequences (結果)
- **メリット**:
  - **高い統制力**: Angular の厳格なディレクトリ構造やモジュール定義により、大規模開発でもコードの品質を一定以上に保ちやすい。
  - **バックエンドとの親和性**: TypeScript のクラスベースの記法や DI の仕組みは Java / Spring Boot と共通点が多く、フルスタックな学習効率が高い。
  - **長期安定性**: 必要な機能がコアライブラリに統合されているため、サードパーティ製ライブラリの選定・更新に伴うリスクが低い。
- **副作用 / 残存リスク**:
  - **学習曲線**: React や Vue.js と比較して概念が多く、習得に時間を要する（特に RxJS や Observable の理解）。
  - **バンドルサイズ**: 小規模なランディングページなどには過剰（オーバーキル）な構成となる。
