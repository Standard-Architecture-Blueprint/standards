# RD-001: 共通非機能要件定義書 (Common Non-Functional Requirements)

## 📌 概要
本ドキュメントは、SABプロジェクト全体における「品質特性」および「技術制約」を定義する。これらは個別の機能要件とは独立し、システム全体の信頼性、性能、セキュリティ等の基準となる。AP試験における「非機能要件グレード」の主要項目に準拠する。

---

## 🏗️ 1. 可用性 (Availability)
システムが継続して稼働し続けるための能力。

| 項目 | 目標値・定義 | 備考 |
| :--- | :--- | :--- |
| 稼働時間 | 24時間365日 | 計画停止を除く |
| 稼働率目標 | 99.9% 以上 | 年間停止時間 8.76時間以内 |
| 目標復旧時間 (RTO) | 2時間以内 | 大規模障害発生時 |
| 目標復旧時点 (RPO) | 24時間前 (前日バックアップ) | |

---

## 🚀 2. 性能・拡張性 (Performance & Scalability)
応答速度および負荷に対する耐性。

| 項目 | 目標値・定義 | 備考 |
| :--- | :--- | :--- |
| 画面応答速度 | 平均 2.0秒以内 (ピーク時 5.0秒以内) | |
| API応答速度 | 平均 500ms以内 | |
| 同時接続ユーザー数 | 最大 500ユーザー (同時実行 50件) | |
| スケール戦略 | 水平スケーリング (リポジトリ単位) | 負荷に応じたPod数の増減 |

---

## 🔒 3. セキュリティ (Security)
資産の保護および不正アクセスの防止。

| 項目 | 定義・採用基準 | 備考 |
| :--- | :--- | :--- |
| 認証方式 | OAuth 2.0 / OpenID Connect | IDP連携 |
| 暗号化 (通信) | TLS 1.3 必須 | |
| 暗号化 (データ) | AES-256 (保存データの暗号化) | DB透過的暗号化 |
| ログ保持 | 操作ログ・アクセスログを1年間保持 | 監査証跡として利用 |

---

## 🛠️ 4. 運用・保守性 (Operability & Maintenance)
システムの維持管理のしやすさ。

| 項目 | 定義・基準 | 備考 |
| :--- | :--- | :--- |
| 監視項目 | 死活監視、リソース監視 (CPU/Mem/Disk) | |
| バックアップ | 1回/日 (フルバックアップ) | 世代管理：7日間 |
| デプロイ方式 | ブルー・グリーンデプロイ / カナリアデプロイ | サービス無停止を原則とする |

---

## 🏗️ 5. 技術制約・基盤要件 (System Constraints)
アーキテクチャ上の物理的・技術的な制約。

*   **インフラ基盤**: クラウドネイティブ環境 (Kubernetes / Docker)
*   **開発言語規格**: Java 17+ / TypeScript (Angular) 
*   **データベース**: PostgreSQL (リレーショナル) / Redis (キャッシュ)
*   **リポジトリ構成**: 13リポジトリ分割構成の維持
