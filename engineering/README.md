# Engineering Standards (開発工学標準)

## 📌 概要
本ディレクトリは、システムを高品質かつ効率的に構築するための「工学的プロセス」を定義する。応用情報技術者試験（AP）におけるソフトウェア開発工程、および現代の DevOps プラクティスに基づいた標準を定める。

## 🎯 目的
1. **再現性の確保**: 開発環境から本番環境まで、一貫した動作を保証する。
2. **継続的品質向上**: テストとレビューの自動化により、デグレードを未然に防ぐ。
3. **チーム開発の最適化**: 複数リポジトリに及ぶ大規模開発において、作業の競合を最小化する。

## 📐 基本エンジニアリング原則
- **Shift Left**: テストやセキュリティチェックを開発の早期段階（左側）で実施する。
- **Automate Everything**: 人の手による反復作業を可能な限り自動化し、ヒューマンエラーを排除する。
- **Infrastructure as Code (IaC)**: 環境構築手順をマニュアル化せず、すべてコードで管理する。

## 🚦 品質ゲート (Quality Gates)
プルリクエストがマージされるためには、以下の基準をすべて満たす必要がある。
1. **Lint/Format**: 静的解析をパスし、命名規則やスタイルが守られていること。
2. **Automated Tests**: すべての単体テストが成功し、カバレッジが目標値を超えていること。
3. **Peer Review**: 少なくとも1名以上のアーキテクトまたはリードエンジニアの承認を得ること。
