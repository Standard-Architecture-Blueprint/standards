# Environment Setup Guide (環境構築標準)

## 1. 📌 概要
本ドキュメントは、本システムの開発に必要なローカル環境および各実行環境の構築手順と標準構成を定義する。複数のリポジトリが相互に連携する構成において、「誰の環境でも同じように動く」状態を最小の工数で実現することを目的とする。

## 2. 💻 ローカル開発環境の要件
応用情報技術者試験の「ハードウェア」および「OS」の概念に基づき、以下のスペックを推奨する。

- **OS**: Windows 11 (WSL2必須), macOS (Sequoia以上), または Linux
- **CPU/メモリ**: 8コア以上 / 16GB以上（複数のマイクロサービスを同時起動するため）
- **ランタイム**:
  - Java 17 (OpenJDK)
  - Node.js 20.x (LTS)
  - Docker / Docker Compose

## 3. 🐳 コンテナ化戦略
環境の差異による「自分の環境では動いた（It works on my machine）」問題を排除するため、すべてのミドルウェアを Docker で管理する。

### 3.1. Docker Compose の活用
- 共通の `docker-compose.yml` を提供し、以下のコンテナを一括で起動可能にする。
  - データベース (PostgreSQL)
  - メッセージブローカー (RabbitMQ / Redis)
  - API ゲートウェイ (認証・認可用 Platform レイヤ)
- **開発モード**: 各リポジトリのソースコードをマウントし、ホットリロードが効く状態で実行する。

## 4. 🔑 設定管理 (Configuration)
AP試験の「情報セキュリティ」における機密管理に基づき、環境変数と設定ファイルを分離する。

- **`.env.example`**: 必要な環境変数のテンプレート。このファイルを `.env` としてコピーして使用する。
- **機密情報の除外**: APIキー、パスワード、秘密鍵などは絶対に Git 管理下（リポジトリ内）に含めない。`.gitignore` で徹底する。
- **プロファイル切り替え**: Spring Boot 等の機能を用い、`local`, `dev`, `prod` の設定を切り替える。

## 5. 🛠️ セットアップ・ステップ
1. **リポジトリのクローン**: 必要なマイクロサービスのリポジトリを所定のディレクトリ構造で配置する。
2. **共通インフラの起動**: `docker-compose up -d` により共通DB等のミドルウェアを起動。
3. **依存ライブラリの解決**: `mvn install` または `npm install` の実行。
4. **初期データの投入**: 標準化されたシードデータ（`design/data-modeling-standards.md` 準拠）をDBへインポート。

## 6. 🌐 ネットワークと接続確認
- **ポート割り当て**: 13のリポジトリが競合しないよう、各サービスのポート番号を `8081` から `8093` の範囲で事前に割り当てる（ポートマップ定義参照）。
- **疎通確認**: 各エンドポイントの `/health` または `/actuator/health` を確認し、サービスが健全であることを検証する。

## 7. 📖 トラブルシューティング
- ログの確認方法（`docker logs` 等）や、よくあるエラー（ポート競合、メモリ不足）への対処法を wiki または本ドキュメントの末尾に集約する。
